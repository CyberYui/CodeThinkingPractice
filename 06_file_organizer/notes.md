# 06 文件整理器 - 笔记

## 我的设计思路
    主程序只做为简单提示和调用函数的入口，实际操作在函数中进行，文件类型统一打包到一个类中，作为一个数据类来存储常用的文件类型和对应的扩展名，方便后续维护和扩展。后面需要使用这些数据的时候，通过一个函数来判断文件类型并创建对应的文件夹，这样可以避免代码重复，提高代码的可读性和可维护性。

## 卡住的地方
    创建的类实际上是个字典，如何解包字典以及内部的列表来进行判断
    使用哪个库来进行文件操作，以及如何动态判断文件类型并移动文件

## 学到的
    1. 使用类来存储相关数据，可以提高代码的组织性和可维护性。
    2. 使用 os 模块来进行文件操作，如创建文件夹、移动文件等。
    3. 使用 os.path.splitext() 来获取文件的扩展名，并进行分类判断。
    4. os.makedirs() 的简单逻辑，并且可以设置 exist_ok=True 来避免重复创建文件夹时抛出异常导致程序结束
    5. 巩固了关于字典解包的知识，特别是在循环中使用 items() 方法来获取键值对进行操作

## 和AI代码的差异
    AI 显然在解包字典和列表上做得更好，使用了 items() 方法来获取键值对进行判断，而我在这方面的思路比较混乱，导致代码不够简洁和高效。此外，AI 还使用了 os.makedirs() 来创建文件夹，并且设置 exist_ok=True 来避免重复创建文件夹时抛出异常，这也是我没有想到的细节。
    AI 的代码帮我解决了重复判断和移动文件的问题，一个函数确定分类，一个函数负责创建文件夹和移动文件，这样的设计使得代码更加清晰和易于维护。
    AI 的代码结构更清晰，职责分明，主函数只负责调用其他函数，而具体的操作逻辑都封装在各自的函数中，这样的设计使得代码更易读和维护。

## 运行结果
    运行结果符合预期，文件被正确地分类并移动到对应的文件夹中。程序能够正确识别不同类型的文件，并且在需要时创建相应的文件夹来存放这些文件。

## 下次改进
    1. 可以添加更多的文件类型和对应的扩展名，以覆盖更多的文件分类。
    2. 可以添加错误处理机制或者日志记录，例如当文件移动失败时，能够捕获异常并给出相应的提示。
    3. 可以添加一个功能来处理重复文件，例如在移动文件时，如果目标文件夹中已经存在同名文件，可以选择重命名新文件或者覆盖旧文件。
    4. 可以添加一个UI界面，让用户更方便地选择文件夹路径和查看分类结果。
